# Task Prompt: 7.1 - Design system and branded help

## Objective

Establish the shared terminal design system for Launchpad so later Phase 7 work
can polish commands against one consistent set of themes, render helpers, help
layout rules, and branding decisions.

## Context

- The current help configuration in `src/launchpad_cli/cli/__init__.py` only
  enables a few default `rich-click` settings; it does not yet define a real
  Launchpad visual identity.
- Shared Rich render helpers live in `src/launchpad_cli/display.py`, but they
  are currently limited to basic tables and panels without a palette, badges,
  or reusable next-step or status blocks.
- The installed environment already includes `rich-click` 1.9.7, so this task
  should stay on the existing `click` + `rich` + `rich-click` stack rather than
  introducing a heavier terminal framework.
- Phase 7 direction is explicit: bold color, guided UX, and a compact ASCII
  Launchpad mark that is suppressed in `--json`, `--quiet`, `--no-color`, and
  non-TTY output.

## In Scope

- Add the shared display/theme layer needed for Phase 7, including palette
  choices, status badges, panel conventions, section helpers, and next-step
  renderables that later command tasks can reuse.
- Rework root and subcommand help so implemented commands share one branded,
  easy-to-scan help layout with grouped sections and examples.
- Introduce a compact ASCII Launchpad wordmark for root help and selected
  human-facing entry surfaces, with explicit suppression in quiet or
  machine-oriented modes.
- Update the relevant help-facing docs and add tests that cover help output and
  no-color behavior.

## Out of Scope

- Redesigning the runtime behavior of `submit`, `status`, `download`, or the
  other workflow commands beyond the shared presentation layer they depend on.
- Adding new persistent theme configuration or new root flags.
- Replacing `click`, `rich`, or `rich-click` with another UI framework.

## Deliverables

- Shared display or theme helpers under `src/launchpad_cli/`
- Root CLI wiring and help configuration updates
- Tests covering branded help and no-color suppression
- User-facing documentation updates for the new help presentation where needed
