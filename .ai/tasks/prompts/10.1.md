# Task 10.1 - Nastran Env Exports and Live Running-Job Metrics

## Objective

Land the deferred cluster-facing follow-up that adds project-configurable
Nastran environment exports and live running-job resource metrics, while
preserving Launchpad behavior on clusters without SLURM accounting.

## Context

- The human explicitly requested that this work be included after noticing it
  was not part of the Phase 9 merge.
- A local prototype already existed, but it was restored before the Phase 9
  merge when the Coordinator role boundary was reasserted.
- The new work should now be landed with a proper `.ai` paper trail.
- Keep the current machine-facing interfaces stable unless the prompt says
  otherwise.

## In Scope

- Add `solvers.nastran.environment` to config and thread it into Nastran submit
  script environment exports.
- Add `remote_binaries.sstat` to config and doctor/docs coverage.
- Add `sstat` command building, parsing, and query helpers for live running-job
  metrics.
- Surface live `sstat` metrics in human-readable `status` output for both
  overview/watch and specific-job detail views.
- Keep `launchpad --json status ...` unchanged in this first pass.
- Degrade gracefully when `sacct` is unavailable or accounting is disabled:
  running-job status should still work, and completed-job metrics/history may
  show limited data instead of failing.
- Update docs and regression coverage for the new behavior.

## Out of Scope

- Changing existing JSON status payloads.
- Adding new solver implementations.
- Adding historical metric backfill for clusters that truly lack accounting.
- Reworking the broader Phase 9 display grammar beyond what the new metrics and
  notices require.

## Deliverables

- Config/model updates for `solvers.nastran.environment` and
  `remote_binaries.sstat`
- Nastran adapter + submit-script environment export support
- Shared SLURM `sstat` helpers and status-command integration
- Doctor degraded-mode handling for optional `sacct` / `sstat`
- Updated docs covering project-local license env config and degraded metrics
- Tests covering config, submit, SLURM helpers, status behavior, and doctor
