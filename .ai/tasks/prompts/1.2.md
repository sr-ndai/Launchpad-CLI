# Task Prompt: 1.2 - Config and logging foundation

## Objective

Implement the layered configuration system and logging bootstrap required for
the CLI to load settings consistently and expose initial config commands.

## Context

- Phase 1 calls for `core/config.py`, `core/logging.py`, and
  `launchpad config init/show`.
- `.ai/plan.md` sections 4.2, 6, 8.1, 10, 12, and 15 define the config model,
  precedence order, logging expectations, and documentation standards.
- Task `1.1` establishes the package, entry points, and test conventions this
  task should build on.

## In Scope

- Implement Pydantic-based config models for the initial Launchpad settings
  described in the plan.
- Load and merge configuration from cluster, user, project-local, environment,
  and CLI layers in the documented precedence order.
- Implement the repository's logging bootstrap with file and console sinks
  appropriate for future commands.
- Add `launchpad config init` and `launchpad config show` with behavior that is
  useful for local setup and inspection.
- Add or expand tests covering config loading, precedence, and logging setup.
- Update docs where needed so the config workflow is discoverable.

## Out of Scope

- SSH connectivity checks, doctor diagnostics, transfer operations, or SLURM
  interactions.
- Full command implementations outside config-facing behavior.

## Deliverables

- `src/launchpad_cli/core/config.py`
- `src/launchpad_cli/core/logging.py`
- Config command wiring and tests/documentation updates
