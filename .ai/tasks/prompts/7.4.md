# Task Prompt: 7.4 - Secondary command polish and docs

## Objective

Finish Phase 7 by bringing the remaining implemented operator commands and the
user-facing docs into the same terminal design language as the primary
workflows.

## Context

- Tasks `7.1` to `7.3` establish the shared design system and apply it to the
  first-run and primary workflow surfaces.
- The remaining implemented commands (`logs`, `ls`, `cancel`, `cleanup`, and
  any small human-facing gaps still left elsewhere) should not keep the old
  look once the main workflows are refreshed.
- Phase 6 already created a substantial docs set, so this task should update
  those docs with current output examples rather than inventing a parallel docs
  structure.
- By the end of this task, every implemented human-facing command should look
  like it belongs to the same CLI.

## In Scope

- Apply the shared Phase 7 design system to `logs`, `ls`, `cancel`, `cleanup`,
  and any remaining implemented command surfaces that still visibly use the old
  style.
- Standardize confirmations, success summaries, empty states, and follow-up
  hints for the touched operator commands.
- Update README and the command-reference docs so examples and descriptive text
  match the refreshed terminal experience.
- Add or update tests for the touched commands.

## Out of Scope

- New commands or new non-terminal UI frameworks
- Changing documented JSON payload shapes
- Fresh-machine rollout validation or shared cluster configuration work

## Deliverables

- Updated secondary-command presentation and operator guidance
- Tests covering the touched command surfaces
- README and command-reference updates that reflect the Phase 7 experience
