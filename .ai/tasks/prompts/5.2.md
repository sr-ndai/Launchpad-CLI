# Task Prompt: 5.2 - Transfer implementation and resume hardening

## Objective

Implement the transfer direction chosen in task `5.1` and harden interrupted
transfer behavior so submit and download remain correct, recoverable, and clear
for operators.

## Context

- Task `5.1` will leave a benchmark document with the chosen transfer
  direction, or an explicit decision to keep the existing single-stream path.
- The current transfer engine lives in `src/launchpad_cli/core/transfer.py`,
  with configuration in `src/launchpad_cli/core/config.py` and integration
  points in `src/launchpad_cli/cli/submit.py` and
  `src/launchpad_cli/cli/download.py`.
- The roadmap still prioritizes correctness, resume support, and integrity over
  raw throughput. Faster is only acceptable if it stays debuggable and
  dependable.

## In Scope

- Implement the chosen transfer direction from `docs/transfer-benchmark.md`, or
  codify the decision to keep single-stream while tightening its failure and
  recovery behavior.
- Update configuration surfaces only where the chosen strategy requires them,
  keeping safe defaults for everyday CLI use.
- Harden interrupted upload/download handling, resume semantics, and operator-
  facing error messages around partial transfers.
- Add or update tests and documentation that cover the selected transfer path
  and its recovery guarantees.

## Out of Scope

- CLI typo suggestions or broad `--json` coverage work assigned to later tasks.
- Logging audit beyond the modules directly touched by the transfer changes.
- Solver-specific feature work.

## Deliverables

- `src/launchpad_cli/core/transfer.py`
- `src/launchpad_cli/core/config.py`
- `src/launchpad_cli/cli/submit.py`
- `src/launchpad_cli/cli/download.py`
- Associated tests and documentation updates
