# Task 11.2 — UI Consistency Sweep

## Objective

Fix all places in the CLI command files that bypass or violate the Phase 9
design system. `display.py` and `__init__.py` are already fully compliant —
only `cli/*.py` command files need changes. Three categories of work: error
message quality, missing progress indicators, and quiet/no-color leaks.

## Context

A post-Phase-9 audit found:

1. **Raw error messages** — Many `raise click.ClickException(message)` calls
   produce error text without actionable suggestions. The design system spec
   (`LAUNCHPAD_UI_REDESIGN_PLAN.md` §2.10) requires every error to suggest a
   next step.

2. **Missing spinners for non-transfer operations** — Six slow remote
   operations run silently:
   - `ls.py`: recursive `find` command during directory listing
   - `cleanup.py`: remote directory discovery and per-directory `du` size loop
   - `logs.py`: initial SSH read before log content appears
   - (submit and download spinners are handled in Task 11.1)

3. **`click.echo("Interrupted.")` quiet-mode leaks** — Three commands use
   `click.echo()` for the Ctrl-C interrupt message instead of `console.print()`.
   This ignores `--no-color` and cannot be suppressed with `--quiet`.

## In Scope

### 1. Error message quality — all `cli/*.py` files

Audit every `raise click.ClickException(...)` call. For each one, check whether
the message already includes an actionable suggestion. If not, append a short
suggestion in parentheses or as a second sentence.

**Pattern to follow — from `doctor.py`:** DiagnosticResults always carry both
a `detail` (what went wrong) and a `suggestion` (what to do). Apply this spirit
to ClickException messages.

**Examples of the change:**

| Before | After |
|--------|-------|
| `"Input directory does not exist: {path}"` | `"Input directory does not exist: {path}. Run from your solver input folder or pass the path explicitly."` |
| `"--watch does not support --json"` | `"--watch does not support --json. Use --json without --watch for a one-shot status snapshot."` |
| `"No SLURM job data found for {job_id}"` | `"No SLURM job data found for {job_id}. Verify the job ID with: launchpad status"` |
| `"Cancellation aborted."` | `"Cancellation aborted."` (no change — no suggestion needed) |

**Do not** redesign error rendering — `click.ClickException` is still the right
mechanism. Only improve the message text.

Files to audit: `submit.py`, `status.py`, `download.py`, `logs.py`, `cancel.py`,
`ls.py`, `cleanup.py`, `config_cmd.py`, `ssh_cmd.py`.

### 2. Missing spinners — `ls.py`, `cleanup.py`, `logs.py`

Add `build_spinner(console, message)` context managers around slow remote
operations. All three files already import `console` and use design primitives.

**`ls.py`** — The `list_remote_directory()` call (and optional `measure_remote_path()`)
runs a remote `find` command. On large directories this can take seconds. Add:
```python
with build_spinner(console, "Listing remote directory..."):
    entries = await list_remote_directory(conn, remote_path, recursive=recursive)
```

**`cleanup.py`** — Two slow operations:
1. Remote directory listing to discover job directories (a `find` call)
2. The loop that calls `measure_remote_path()` for each directory's size (`du`)

Wrap each with a spinner. For the size loop, a single spinner is sufficient
("Measuring sizes...") since the individual `du` calls are fast and the total
count is shown in the summary.

**`logs.py`** — The initial `_fetch_log_head()` or equivalent SSH read that
happens before log content starts streaming. Add a spinner with
"Reading log..." that clears when content appears.

### 3. `click.echo("Interrupted.")` → `console.print(...)` — 3 files

Replace with a `console.print()` call that respects the theme and quiet mode.
Use the existing `build_status_line` or just a plain `console.print("Interrupted.")`.

Files:
- `status.py` — approximately line 207
- `logs.py` — approximately line 173
- `download.py` — approximately line 293

**Pattern:**
```python
# BEFORE
click.echo("Interrupted.")

# AFTER
console.print("  Interrupted.", style="lp.text.tertiary")
```

### 4. Help text completeness

Scan every Click command and option for:
- Any `@click.option(...)` without a `help=` string → add one
- Any `@click.command(...)` without a `help=` docstring body → add one

This is a quick scan; most commands are already well-documented.

## Out of Scope

- Changes to `display.py` (already compliant).
- Changes to `__init__.py` rich-click config (already correct).
- Redesigning any command's visual layout.
- JSON output changes.
- Adding `--quiet` suppression to log-streaming content (intentional — users
  need to see log content).
