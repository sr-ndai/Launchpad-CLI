# Task Prompt: 4.2 - Download command orchestration

## Objective

Implement the full `launchpad download` experience for completed job results on
top of the established Phase 4 primitives.

## Context

- Phase 4’s main user-facing milestone is that engineers can retrieve results
  safely from the local CLI after submission and monitoring.
- `.ai/plan.md` sections 3.4, 4.2, 4.4, 5.1, 8, 12, and 15 describe the
  intended download flow, command behavior, verification expectations, and
  documentation standard.
- This task depends on `4.1` being complete and merged.

## In Scope

- Replace the placeholder `launchpad download` command with the Phase 4
  retrieval flow: job lookup/status gating, remote size estimation, local
  free-space check, confirmation, remote compression policy, transfer,
  decompression, and verification.
- Support the documented Phase 4 command surface needed now, including
  destination selection, cleanup-on-success, force/space override, remote
  compression policy, task selection, exclude/include controls, and compression
  level.
- Add focused tests for command wiring and deterministic orchestration using
  fakes for remote effects.
- Update documentation for the download command.

## Out of Scope

- `launchpad ls` and `launchpad cleanup` command wiring.
- Parallel multi-stream transfer work or broader Phase 5 polish.

## Deliverables

- `src/launchpad_cli/cli/download.py`
- Associated tests and documentation updates

`task/4.2-download-command-orchestration`
