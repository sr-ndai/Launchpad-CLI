# Task Prompt: 3.2 - Status command and live monitoring UI

## Objective

Implement the first functional `launchpad status` experience, including Rich
status presentation and `--watch` polling.

## Context

- Phase 3’s central user-facing milestone is that engineers can monitor jobs
  entirely from the local terminal.
- `.ai/plan.md` sections 3.2, 4.2, 7.2, 8, 12, 14, and 15 describe the intended
  status command behavior, live refresh model, and documentation expectations.
- This task depends on `3.1` being complete and merged.

## In Scope

- Replace the placeholder `launchpad status` command with the Phase 3 status
  flow for current-user and specific-job queries.
- Implement `--watch` and `--interval` behavior around the established SLURM
  query primitives.
- Add Rich-facing status rendering in `display.py` as needed for summary and
  per-task tables.
- Add tests for command wiring and any deterministic status formatting or watch
  logic that can be validated with fakes.
- Update documentation for the status command.

## Out of Scope

- Remote log viewing, cancellation, download, or cleanup workflows.
- Expanding JSON support beyond what the prompt requires.

## Deliverables

- `src/launchpad_cli/cli/status.py`
- `src/launchpad_cli/display.py`
- Associated tests and documentation updates
