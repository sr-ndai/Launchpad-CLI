# Task Prompt: 8.7 - Syntax-highlighted config show output

## Objective

Improve `launchpad config show` by rendering the human-readable resolved TOML
with real syntax highlighting instead of plain text.

## Context

- The active coordination branch is
  `phase/08-task-references-and-solver-aware-logs`, and PR `#10` is open.
- The current `config show` command still emits the resolved TOML through
  `click.echo(dumps_toml(...))` in `src/launchpad_cli/cli/config_cmd.py`, so
  the primary human config-inspection path has no Rich syntax highlighting.
- The rest of the CLI already uses the shared Rich console and themed output
  helpers, so this plain-text path now stands out as inconsistent.
- `launchpad --json config show` and `launchpad config show --docs` are already
  useful as machine-readable and copy/paste-friendly outputs and should remain
  unchanged.

## In Scope

- Update the non-JSON, non-`--docs` `launchpad config show` path to render the
  resolved TOML with Rich syntax highlighting.
- Use the existing console/no-color behavior so `--no-color` and `NO_COLOR`
  still degrade cleanly.
- Keep the resolved config content and TOML serialization unchanged; this task
  is about presentation, not config semantics.
- Preserve `launchpad --json config show` output exactly as JSON.
- Preserve `launchpad config show --docs` output as the plain documented schema.
- Add or update regression tests covering the human-readable `config show`
  behavior plus the unchanged `--json` / `--docs` branches where needed.
- Update the relevant docs to mention that `config show` renders the resolved
  TOML with syntax highlighting in human-readable mode.

## Out of Scope

- Changes to config resolution, precedence, or TOML serialization
- New config keys, environment aliases, or `config init` changes
- Broad terminal-theme refactors beyond what is needed for the highlighted
  `config show` surface
- Changes to `--json` or `--docs` formatting

## Deliverables

- Syntax-highlighted human-readable `launchpad config show`
- Regression coverage for the updated rendering path
- Docs updates describing the highlighted output behavior
