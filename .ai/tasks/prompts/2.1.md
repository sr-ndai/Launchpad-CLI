# Task Prompt: 2.1 - Solver adapters and input discovery

## Objective

Implement the solver abstraction layer required for Phase 2 so the submit
pipeline has concrete, testable Nastran behavior and an explicit ANSYS stub.

## Context

- Phase 2 begins with the solver contract because `launchpad submit`,
  `core/slurm.py`, and later status/logs work all depend on stable adapter
  behavior.
- `.ai/plan.md` sections 3.1, 4.2, 4.3, 7.1, 12, and 15 describe the intended
  submit flow, module responsibilities, testing expectations, and
  documentation standard.
- The current repository still has placeholder solver modules from Phase 1.

## In Scope

- Expand `solvers/base.py` into the Phase 2 adapter protocol expected by the
  submit flow.
- Implement the Nastran adapter behavior needed now: input discovery,
  validation-friendly metadata, run command construction, and scratch
  environment setup.
- Keep the ANSYS adapter as a Protocol-compliant stub that fails clearly for
  unsupported runtime behavior.
- Add focused tests that cover the solver protocol and discovery behavior
  without requiring SSH or a real cluster.
- Update documentation where the solver layer is described.

## Out of Scope

- Remote filesystem helpers, SLURM submission, or the `launchpad submit`
  command itself.
- Automatic INCLUDE parsing or any ANSYS workflow implementation beyond the
  documented stub.

## Deliverables

- `src/launchpad_cli/solvers/base.py`
- `src/launchpad_cli/solvers/nastran.py`
- `src/launchpad_cli/solvers/ansys.py`
- Associated tests and documentation updates
