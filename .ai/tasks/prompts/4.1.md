# Task Prompt: 4.1 - Download primitives and filesystem groundwork

## Objective

Implement the reusable local, remote, and archive helpers required for Phase 4
result retrieval and remote filesystem management.

## Context

- Phase 4 starts with core primitives because `launchpad download`,
  `launchpad ls`, and `launchpad cleanup` all depend on reliable disk-space
  checks, remote archive handling, and remote filesystem metadata.
- `.ai/plan.md` sections 3.4, 3.6, 3.9, 4.2, 4.4, 5.1, 12, and 15 describe the
  intended download flow, filesystem responsibilities, testing expectations,
  and documentation standard.
- The current repository already includes placeholder Phase 4 commands plus the
  existing local compression and single-stream transfer primitives from earlier
  phases.

## In Scope

- Extend the local filesystem helpers with the Phase 4 disk-space and
  destination-path behavior needed for download workflows on Windows-first
  machines.
- Extend the remote filesystem and compression helpers with the Phase 4
  archive, size, checksum/introspection, list, and delete operations needed by
  the later commands.
- Add focused tests for the new core behavior using fakes rather than a live
  cluster.
- Update documentation where the download and remote filesystem layers are
  described.

## Out of Scope

- CLI orchestration for `launchpad download`, `launchpad ls`, or
  `launchpad cleanup`.
- Parallel transfer optimization or Phase 5 resume hardening.

## Deliverables

- `src/launchpad_cli/core/local_ops.py`
- `src/launchpad_cli/core/remote_ops.py`
- `src/launchpad_cli/core/compress.py`
- Associated tests and documentation updates
