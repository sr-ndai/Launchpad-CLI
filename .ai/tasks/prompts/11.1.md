# Task 11.1 — Transfer Progress Indicators

## Objective

Close all silent long-running I/O gaps in `lp submit` and `lp download` by
wiring spinners and progress bars using the Rich infrastructure that already
exists in `display.py`. No new dependencies, no new display primitives — only
missing call-sites.

## Context

`lp download` already shows:
- A spinner while remote compression runs (`download.py:510`).
- A Rich progress bar during the transfer (`download.py:545`, `download.py:647`).

`lp submit` shows nothing at any step. `lp download`'s local decompression is
also silent. This task fills those gaps.

## Gap Table

| Operation | Current | Target |
|-----------|---------|--------|
| Submit: local compress | Silent (blocks event loop) | Spinner + `asyncio.to_thread()` |
| Submit: upload single-file | Silent | Determinate progress bar |
| Submit: upload multi-file | Silent | Determinate progress bar |
| Submit: remote extract | Silent | Spinner |
| Download: remote compress | ✅ Spinner | No change |
| Download: transfer | ✅ Progress bar | No change |
| Download: local decompress | Silent (blocks event loop) | Spinner + `asyncio.to_thread()` |

## In Scope

1. **`src/launchpad_cli/core/transfer.py`**
   - Add `progress_callback: ProgressCallback | None = None` to `upload_many()`
     (currently at line 196). Mirror the `download_many()` implementation
     exactly: add a `progress_by_item: dict[str, int] = {}` accumulator and a
     per-worker `on_progress` closure that emits cumulative bytes via
     `_emit_progress(progress_callback, transferred_total)`.
   - Add `progress_callback: ProgressCallback | None = None` to
     `striped_upload()` (currently at line 293) and thread it through to the
     internal `upload_many()` call.

2. **`src/launchpad_cli/cli/submit.py`**
   - Add `console` parameter to `_execute_submit(plan)`. Update the single call
     site (`asyncio.run(_execute_submit(plan))` at line 291) to pass the
     `console` object created at line 258.
   - **Local compression**: replace the synchronous `_materialize_archive(...)`
     call with `await asyncio.to_thread(_materialize_archive, ...)` wrapped in
     `build_spinner(console, "Compressing...")`. This keeps the event loop free
     so the spinner animates.
   - **Upload (single-file)**: replace the bare `striped_upload(...)` call with
     `_run_with_upload_progress(console, total_bytes=archive_path.stat().st_size,
     description="Uploading", operation=lambda cb: striped_upload(...,
     progress_callback=cb))`.
   - **Upload (multi-file)**: replace the bare `upload_many(...)` call with
     `_run_with_upload_progress(console, total_bytes=sum(item.local_path.stat().st_size
     for item in upload_items), description="Uploading",
     operation=lambda cb: upload_many(..., progress_callback=cb))`.
   - **Remote extraction**: wrap `extract_remote_archive(...)` with
     `build_spinner(console, "Extracting on cluster...")`.
   - **Add helper** `_run_with_upload_progress(console, *, total_bytes, description,
     operation)` — identical structure to `_run_with_download_progress` in
     `download.py:1025`. If `total_bytes <= 0`, call `await operation(None)`.
     Otherwise open `build_progress(console=console)`, add a task, build an
     `on_progress` closure, and `await operation(on_progress)`.
   - Add `asyncio` to imports (if not already present). Add `build_progress`,
     `build_spinner` to the `launchpad_cli.display` imports.

3. **`src/launchpad_cli/cli/download.py`**
   - **Local decompression** (currently line 571): replace the bare
     `decompress_path(...)` call with:
     ```python
     with build_spinner(console, "Extracting..."):
         await asyncio.to_thread(decompress_path, plan.local_archive_path, plan.destination_dir)
     ```
   - Add `asyncio` to imports (if not already present).

## Out of Scope

- Per-stream breakdown or per-file progress views.
- Checksum verification progress.
- Changes to the existing download remote-compress spinner or download progress bar.
- Any new display primitives beyond `build_progress` and `build_spinner`.
- JSON output path changes.
