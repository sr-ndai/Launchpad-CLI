# Task Prompt: 5.3 - CLI polish and logging audit

## Objective

Finish the remaining Phase 5 CLI polish by closing JSON and UX gaps, handling
documented operator edge cases, and auditing logging behavior across the
user-facing commands.

## Context

- The root CLI currently exposes global `--json`, `--verbose`, and `--no-color`
  flags, but structured JSON output is still inconsistent across commands.
- `src/launchpad_cli/cli/__init__.py` does not yet provide typo suggestions for
  invalid subcommands.
- Logging is centralized in `src/launchpad_cli/core/logging.py`; the remaining
  work is to make command-level logging and `--verbose` output useful without
  burying operators in noise.

## In Scope

- Add typo suggestions for invalid subcommands in the root Click application.
- Add structured `--json` output to the remaining applicable user-facing
  commands: `submit`, `logs`, `ls`, `cleanup`, and `cancel`.
- Handle the documented operator edge cases around existing directories,
  partial results or transfers, and cancelled jobs in the relevant command
  flows.
- Audit logging calls and verbose behavior in touched modules, adjusting log
  levels and messages where needed while preserving file rotation.
- Add or update tests and user-facing documentation for the polished behavior.

## Out of Scope

- Replacing the transfer strategy again after task `5.2` lands.
- `solvers/ansys.py` implementation.
- Reworking the raw SSH passthrough command into a structured JSON surface.

## Deliverables

- `src/launchpad_cli/cli/__init__.py`
- Relevant CLI modules needed for the remaining JSON and edge-case work
- Logging-related documentation and tests covering the touched commands
