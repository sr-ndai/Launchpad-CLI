# Task Prompt: 9.2 - Onboarding, config, and diagnostics redesign

## Objective

Move `launchpad config init`, the human-readable default `launchpad config show`
path, and `launchpad doctor` onto the Phase 9 restrained visual grammar while
preserving their existing non-human-facing behavior.

## Context

- The active coordination branch is `phase/09-cli-visual-overhaul`.
- Task `9.1` is merged and provides the new shared display primitives plus the
  restrained bare `launchpad` welcome/help shell in `src/launchpad_cli/display.py`
  and `src/launchpad_cli/cli/__init__.py`.
- The Phase 9 direction keeps branding only on bare `launchpad`. That means
  `config init` and `doctor` must drop the Phase 7 wordmark/success-chrome,
  even on their happy paths.
- Human-readable `launchpad config show` must now stop defaulting to the Rich
  TOML syntax block added in `8.7` and instead render the grouped sectioned
  summary chosen for Phase 9. `--json` and `--docs` stay unchanged.
- Today these surfaces still rely on the older Phase 7 panel stack:
  `config init` uses guided setup and success panels, `doctor` uses summary and
  per-section panels, and `config show` still emits syntax-highlighted TOML for
  the primary human path.
- This task should use the new shared primitives from `display.py` rather than
  introducing more command-local presentation helpers.

## In Scope

- Redesign the human-readable `config init` intro and success paths to use the
  new restrained section/rule, key-value, status-line, and next-step grammar.
- Replace the default human-readable `config show` TOML block with the new
  grouped summary view while preserving the existing resolved data content,
  `--json`, and `--docs`.
- Redesign `doctor` to use the new flattened diagnostics layout, inline status
  lines, restrained summary, and lightweight next steps.
- Remove Phase 7 welcome-style wordmark usage from `config init` and `doctor`
  so bare `launchpad` remains the only branded wordmark surface.
- Update or add regression coverage for the new human-readable output paths.

## Out of Scope

- Redesigning `submit`, `status`, `download`, `logs`, `ls`, `cancel`, or
  `cleanup`
- Changing configuration semantics, precedence, TOML serialization, or doctor
  check logic beyond presentation-layer needs
- Changing `launchpad config show --json` or `launchpad config show --docs`
- README or docs refresh beyond code comments or inline helper notes needed to
  support the implementation

## Deliverables

- Restrained human-readable `config init`
- Sectioned-summary default human-readable `config show`
- Flattened Phase 9 `doctor` output
- Regression coverage for the updated command surfaces
