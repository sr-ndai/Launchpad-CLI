# Task Prompt: 7.3 - Primary workflow terminal experience

## Objective

Apply the Phase 7 design system to Launchpad's highest-value workflows so
`submit`, `status`, and `download` feel cohesive, modern, and helpful during
the moments operators spend the most time in.

## Context

- Task `7.1` establishes the shared theming and help conventions that this task
  should consume instead of inventing command-specific styles.
- Phase 7 feedback explicitly moved personality out of help chrome and into the
  verbs themselves, so this task is where Launchpad should feel most distinct.
- The current human-facing output for `submit`, `status`, and `download` is
  functional but still baseline: simple panels and tables without a stronger
  hierarchy, branded success treatment, or consistent next-step guidance.
- These three commands carry the most visible terminal experience burden in the
  product, especially `submit --dry-run`, `status --watch`, and the download
  confirmation and completion flows.
- Existing `--json` tests already cover these commands and must stay valid.

## In Scope

- Redesign the human-facing output for `launchpad submit`, `launchpad status`,
  and `launchpad download` around the shared Phase 7 display system.
- Improve dry-run, confirmation, empty-state, success, and next-step messaging
  for the touched commands without changing their documented flags.
- Make `launchpad submit` the clearest example of the new runtime personality:
  concise confidence, prominent job ID, and an obvious next command to run.
- Refine `launchpad status --watch` so it gets a cleaner live header and
  updated hierarchy without reprinting decorative branding every refresh.
- Preserve existing JSON output shapes and add or update tests and docs for the
  refreshed workflow experience.

## Out of Scope

- Polishing `logs`, `ls`, `cancel`, or `cleanup`; that belongs to task `7.4`
- Adding new flags or changing existing JSON schemas
- Reworking transfer architecture, SLURM parsing, or other core runtime logic
  unrelated to the human-facing presentation

## Deliverables

- Updated primary-workflow renderers and command wiring
- Tests covering the revised human output while preserving JSON behavior
- Documentation updates for the refreshed `submit`, `status`, and `download`
  experience
