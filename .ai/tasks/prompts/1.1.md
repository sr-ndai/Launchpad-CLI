# Task Prompt: 1.1 - Project scaffolding and baseline CLI

## Objective

Replace the placeholder repository structure with the planned Python package,
entry points, test harness, and documentation skeleton so later Phase 1 work
has a stable foundation.

## Context

- The repository currently contains only a minimal `pyproject.toml`,
  `hello.py`, and an empty `README.md`.
- Phase 1 in `.ai/plan.md` calls for project scaffolding, baseline tests,
  README setup, and an `ARCHITECTURE.md` map before deeper implementation.
- The target package layout and dependency baseline are described in
  `.ai/plan.md` sections 4.1, 9, 10, 12, and 15.
- Runtime target is Python 3.12+ with `uv` as the development workflow.

## In Scope

- Update packaging metadata to match the plan:
  `launchpad-cli`, Python 3.12+, planned runtime dependencies, and `uv`
  development dependencies.
- Replace `hello.py` with a `src/launchpad_cli/` layout that includes:
  package init, a root CLI entry point, placeholder command modules or command
  registration needed for future Phase 1 work, and clear module docstrings.
- Register both console entry points:
  `launchpad` and `lp`.
- Establish the initial test layout under `tests/`, including at minimum CLI
  smoke coverage for the root command and aliases.
- Add `ARCHITECTURE.md` and a non-empty README skeleton aligned with the
  current product plan and development workflow.
- Update `.python-version` and any other repository-level scaffolding needed to
  support the new baseline.

## Out of Scope

- Implementing real config merge logic, logging setup, SSH behavior, transfer
  behavior, compression logic, or SLURM integration.
- Implementing full command behavior for `submit`, `status`, `logs`,
  `download`, `cancel`, `ls`, `config`, `ssh`, `cleanup`, or `doctor`.
- Any production behavior that depends on remote cluster access.

## Deliverables

- A working `src/launchpad_cli/` package with both CLI entry points resolving
  to the root command.
- Updated packaging metadata and repository scaffolding consistent with the
  Phase 1 plan.
- Initial tests that validate the CLI boots and basic project structure is in
  place.
- `README.md` and `ARCHITECTURE.md` skeletons that future tasks can extend.
