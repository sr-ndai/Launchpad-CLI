# Task Prompt: 2.2 - SLURM and remote submission primitives

## Objective

Implement the reusable remote operations and SLURM submission helpers required
by the Phase 2 submit workflow.

## Context

- Phase 2 calls for reusable submit primitives before the CLI orchestration is
  wired so the command layer can stay thin and testable.
- `.ai/plan.md` sections 3.1, 4.2, 4.3, 7.1, 7.3, 8, 12, and 15 describe the
  submit sequence, SLURM job tracking model, error posture, and testing
  expectations.
- This task depends on `2.1` being complete and merged.

## In Scope

- Implement `core/remote_ops.py` helpers for remote directory setup and the
  archive-oriented operations needed by submit.
- Implement `core/slurm.py` script generation and submit wrappers using the
  established solver/config abstractions.
- Add tests for generated SLURM content and the remote helper behavior that can
  be validated with fakes or mocks.
- Update documentation for the new submit primitives.

## Out of Scope

- The `launchpad submit` command wiring and Rich presentation layer.
- Status, logs, download, cleanup, or remote listing workflows beyond what
  submit needs.

## Deliverables

- `src/launchpad_cli/core/remote_ops.py`
- `src/launchpad_cli/core/slurm.py`
- Associated tests and documentation updates
