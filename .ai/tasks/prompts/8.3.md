# Task Prompt: 8.3 - Interactive log picker and docs

## Objective

Finish Phase 8 by turning ambiguous human `logs` usage into a guided task
selection flow and updating the docs/help text to explain the new task-reference
and solver-log model.

## Context

- Tasks `8.1` and `8.2` establish the job manifest, task selectors, and
  solver-aware log lookup.
- The user explicitly chose a lightweight dependency-backed picker for arrow-key
  interaction rather than a custom mini-TUI or a plain numeric prompt.
- The picker should feel consistent with the Phase 7 terminal shell and should
  not leak into `--json` or non-TTY paths.
- Relevant code paths include `src/launchpad_cli/cli/logs.py`,
  `src/launchpad_cli/display.py`, command-reference docs, and any dependency or
  helper wiring added in `8.2`.

## In Scope

- Add the chosen lightweight interactive dependency for task selection.
- When `launchpad logs` targets a multi-task job without `TASK_REF` in human
  TTY mode, open a themed picker instead of hard-failing.
- Show label, alias, raw task ID, task state, selected log kind, and resolved
  filename in the picker rows.
- Ensure follow mode uses retry-by-name semantics so selected logs can be
  followed even if the file appears shortly after selection.
- Keep non-TTY and `--json` behavior deterministic and non-interactive.
- Update relevant help/docs text to explain task references, `--log-kind`, and
  the picker behavior.
- Add or update regression tests for the picker gating and the final `logs`
  behavior.

## Out of Scope

- New interactive surfaces outside `launchpad logs`
- Changing the manifest contract settled in `8.1`
- Retrofitting historical jobs with metadata they never submitted

## Deliverables

- A Phase 7-styled interactive picker for ambiguous human `logs` flows
- Help and docs updates that explain the final Phase 8 behavior
- Regression coverage for picker gating and follow-mode behavior
