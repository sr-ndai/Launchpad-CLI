# Task Prompt: 8.6 - Configurable remote workspace root

## Objective

Fix the remaining field-reported remote-root regression by making Launchpad's
writable shared workspace configurable instead of always deriving it from
`<cluster.shared_root>/<ssh.username>`.

## Context

- The active coordination branch is
  `phase/08-task-references-and-solver-aware-logs`, and PR `#10` is open.
- Field debugging showed that the cluster's writable shared directory is
  `/shared/launchpad`, while Launchpad currently defaults the writable remote
  root to `/shared/<ssh.username>` and therefore points doctor and other
  workspace-oriented commands at a non-existent directory such as
  `/shared/ubuntu`.
- The current derivation appears in at least these production paths:
  `src/launchpad_cli/cli/submit.py`, `src/launchpad_cli/cli/doctor.py`,
  `src/launchpad_cli/cli/ls.py`, and `src/launchpad_cli/cli/cleanup.py`.
- `cluster.shared_root` still represents the shared filesystem mount root and
  should not be repurposed as the Launchpad workspace directory. Operators need
  a separate config field they can set to a writable subtree like
  `/shared/launchpad`.

## In Scope

- Add a dedicated config field for Launchpad's writable remote workspace root
  under `cluster.*`, and expose it through `launchpad config show --docs`.
- Keep `cluster.shared_root` as the shared filesystem root; do not overload it
  with the new workspace meaning.
- Centralize remote workspace-root resolution in shared code so the same logic
  is reused by all commands which currently derive `<shared_root>/<username>`.
- Use that shared workspace-root resolution for:
  - submit remote job-directory construction
  - doctor writable-root checks
  - `launchpad ls` default-root and relative-path resolution
  - `launchpad cleanup` discovery when no job IDs are supplied
- Preserve backward compatibility by falling back to the legacy
  `<shared_root>/<ssh.username>` behavior when the new config field is unset.
- Update operator messaging so doctor failures clearly mention the configured
  workspace root and point users toward the new config key when the cluster's
  writable directory differs from the SSH login username.
- Add or update regression tests for config docs, workspace-root resolution,
  and the affected operator commands.
- Update the relevant docs to explain the distinction between `shared_root` and
  the Launchpad workspace root, including an example such as
  `/shared/launchpad`.

## Out of Scope

- New CLI flags, new short `LAUNCHPAD_*` aliases, or changes to `config init`
  prompts
- Changes to scheduler-shell behavior from `8.5`
- Broad refactors outside the remote workspace-root call sites listed above
- Migration of existing submitted job metadata beyond using the resolved
  workspace root for new operations

## Deliverables

- Dedicated configurable workspace-root field under `cluster.*`
- Shared workspace-root resolver reused by submit, doctor, ls, and cleanup
- Regression coverage for configured and fallback workspace-root behavior
- Docs updates describing the new workspace-root model and troubleshooting path
