# Task Prompt: 9.1 - Display foundation and root shell

## Objective

Build the shared Phase 9 display foundation and migrate the bare `launchpad`
welcome plus the root help shell to the new CLI visual-overhaul direction
without breaking the still-unmigrated command surfaces.

## Context

- The active coordination branch is `phase/09-cli-visual-overhaul`.
- Phase 8 is already merged on `main`, and Phase 9 intentionally supersedes
  much of the current Phase 7 panel-heavy presentation.
- The human chose a full CLI redesign, but with hybrid branding:
  bare `launchpad` remains the only wordmark-capable surface while all other
  surfaces move to the restrained design language.
- Human-readable `launchpad config show` will later move from the current Rich
  TOML block to a sectioned summary, but that is **not** part of this task.
- Today `src/launchpad_cli/display.py` still centers the old badge/panel model
  and already owns shared helpers used by `submit` and `status`, while other
  commands still have command-local renderers. This task must create the new
  shared primitives without forcing all later command migrations into the same
  change.
- The detailed design brief lives at
  `.ai/plans/LAUNCHPAD_UI_REDESIGN_PLAN.md`, and the canonical phase plan for
  sequencing is `.ai/plans/09-cli-visual-overhaul.md`.

## In Scope

- Rework `src/launchpad_cli/display.py` around the new semantic primitives for
  section headers, inline status lines, aligned key/value rows, borderless
  tables, next-step lists, one-hero-panel-max rendering, restrained error
  blocks, and no-color symbol/text fallbacks.
- Add the minimal shared progress/spinner/prompt helper surface that later
  Phase 9 tasks can reuse, but do not migrate the workflow commands onto those
  helpers yet.
- Update the bare `launchpad` welcome and root help shell in
  `src/launchpad_cli/cli/__init__.py` so welcome remains the only branded
  wordmark surface and root help stays grouped but restrained.
- Preserve existing command names, flags, exit codes, JSON payloads,
  `--quiet`, `--no-color`, and non-TTY behavior.
- Keep compatibility for still-unmigrated commands: if old helper names are
  still imported elsewhere, provide a transitional path instead of breaking the
  current command modules in this task.
- Add or update tests covering the new root welcome/help behavior and the new
  shared display semantics that can be exercised safely at this stage.

## Out of Scope

- Redesigning `config init`, `config show`, `doctor`, `submit`, `status`,
  `download`, `logs`, `ls`, `cancel`, or `cleanup` output beyond the shared
  helper and root-shell changes needed to keep the app running
- Updating docs or README examples for the later command-specific redesigns
- Changing any scheduler, transfer, solver, or workspace behavior
- Replacing `click`, `rich`, or `rich-click`, or adding a TUI framework

## Deliverables

- Updated shared display foundation in `src/launchpad_cli/display.py`
- Updated bare `launchpad` welcome and root help shell in
  `src/launchpad_cli/cli/__init__.py`
- Transitional compatibility for existing command modules that still rely on
  pre-Phase-9 display helpers
- Regression coverage for the new root-shell behavior and shared helper
  semantics touched by this task
