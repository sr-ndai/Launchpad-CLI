# Task Prompt: 1.3 - SSH, transfer, and compression primitives

## Objective

Implement the core transport primitives for SSH connectivity, single-stream
file transfer, and local zstd compression/decompression.

## Context

- Phase 1 calls for `core/ssh.py`, `core/transfer.py`, and `core/compress.py`.
- `.ai/plan.md` sections 4.2, 5, 8, 9, 12, and 14 define the intended async
  approach, reliability requirements, and testing posture.
- Task `1.1` provides the package structure this task should extend.

## In Scope

- Implement async SSH connection management using `asyncssh`.
- Implement single-stream SFTP upload/download helpers with progress hooks that
  can later be surfaced through Rich.
- Implement local compression and decompression helpers using `zstandard`.
- Add tests for compression and any transport logic that can be validated
  without requiring the real cluster.
- Document limitations or manual verification steps where local automation is
  impractical.

## Out of Scope

- Parallel transfer benchmarking or multi-stream transfer behavior.
- `submit`, `download`, `doctor`, or `ssh` command orchestration.
- Remote SLURM operations and solver-specific behavior.

## Deliverables

- `src/launchpad_cli/core/ssh.py`
- `src/launchpad_cli/core/transfer.py`
- `src/launchpad_cli/core/compress.py`
- Supporting tests and any required fixture data

## Branch

`task/1.3-ssh-transfer-compression`

## Verification

- `uv run pytest tests/test_compress.py`
- `uv run pytest`
- Manual note in the Builder session covering any transport verification that
  could not be automated locally
