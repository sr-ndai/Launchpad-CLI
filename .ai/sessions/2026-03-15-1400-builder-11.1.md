# Builder Session - 11.1 - 2026-03-15

## Objective
Close all silent long-running I/O gaps in `lp submit` and `lp download` by
wiring spinners and progress bars using existing Rich infrastructure in
`display.py`.

## Outcome
READY_FOR_REVIEW

## What Was Done

### `src/launchpad_cli/core/transfer.py`
- Added `progress_callback: ProgressCallback | None = None` to `upload_many()`.
  Mirrors `download_many()` exactly: `progress_by_item` dict accumulator, per-
  worker `on_progress` closure emitting cumulative bytes via `_emit_progress`.
- Added `progress_callback: ProgressCallback | None = None` to `striped_upload()`
  and threaded it through to the internal `upload_many()` call.

### `src/launchpad_cli/cli/submit.py`
- Added `console` parameter to `_execute_submit(plan, console)`.
- Updated the single call site (`asyncio.run(_execute_submit(plan, console))`)
  to pass the `console` object created at line 258.
- Added `build_progress`, `build_spinner` to display imports.
- Local compression: replaced synchronous `_materialize_archive(...)` with
  `await asyncio.to_thread(_materialize_archive, ...)` wrapped in
  `with build_spinner(console, "Compressing...")`.
- Upload (single-file): replaced bare `striped_upload(...)` with
  `_run_with_upload_progress(console, ...)`.
- Upload (multi-file): replaced bare `upload_many(...)` with
  `_run_with_upload_progress(console, ...)`.
- Remote extraction: wrapped `extract_remote_archive(...)` with
  `with build_spinner(console, "Extracting on cluster...")`.
- Added `_run_with_upload_progress()` helper — identical structure to
  `_run_with_download_progress` in `download.py`.

### `src/launchpad_cli/cli/download.py`
- Local decompression (line 571): replaced bare `decompress_path(...)` with
  `with build_spinner(console, "Extracting..."): await asyncio.to_thread(...)`.
  `asyncio` was already imported.

### `tests/test_submit.py`
- Updated all `fake_execute_submit` stubs (3 occurrences) and 2 direct
  `_execute_submit(plan)` calls to match the new `(plan, console)` signature.
