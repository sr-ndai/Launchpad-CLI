## Observations

- The most conservative interpretation of solver-log viewing was to derive the
  expected file path from the task work directory and the known solver output
  extension, without introducing a broader remote file-listing workflow.
- `launchpad logs` requires an explicit `TASK_ID` when a job exposes multiple
  task rows, rather than guessing which task log the operator intended to view.
- `scancel` currently uses the default remote binary name directly because the
  existing config model does not yet expose a `remote_binaries.scancel` setting.
