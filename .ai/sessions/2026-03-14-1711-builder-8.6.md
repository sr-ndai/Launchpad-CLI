# Builder Session - 8.6 - 2026-03-14 1711

## Objective

Fix the remote workspace-root regression by adding a dedicated configurable
Launchpad workspace root, centralizing its resolution, and applying that shared
resolver to submit, doctor, ls, and cleanup while preserving the legacy
`<shared_root>/<ssh.username>` fallback.

## Outcome

Outcome: READY_FOR_REVIEW

## What Was Done

- Added `cluster.workspace_root` to the config schema and surfaced it through
  the generated config docs.
- Introduced a shared remote workspace-root resolver in
  `src/launchpad_cli/core/workspace.py` which prefers `cluster.workspace_root`
  and otherwise falls back to `<cluster.shared_root>/<ssh.username>`.
- Switched submit, doctor's writable-root check, `launchpad ls`, and
  `launchpad cleanup` discovery to that shared resolver so they all use the
  same workspace path logic.
- Updated doctor messaging so writable-root failures now name the resolved
  workspace path and point operators toward `cluster.workspace_root` when the
  writable cluster directory differs from the SSH login username.
- Expanded regression coverage for config docs, configured and fallback
  resolver behavior, submit path planning, ls base resolution, cleanup
  discovery, and doctor's writable-root check.
- Refreshed the relevant command/config/troubleshooting docs to explain the
  distinction between `shared_root` and the writable Launchpad workspace root.
