# Builder Session - 8.4 - 2026-03-14 1558

## Objective

Fix the Phase 8 cluster-access regressions by making `launchpad doctor` probe
the same remote exec environment Launchpad uses in production and by making
`launchpad ssh` work on Windows through the local OpenSSH client.

## Outcome

Outcome: READY_FOR_REVIEW

## What Was Done

- Changed doctor's remote binary and writable-root checks to execute through
  the normal remote command path instead of forcing `/bin/sh -lc`.
- Tightened the remote-binary failure text so it explains the non-interactive
  SSH exec PATH boundary and points operators toward absolute
  `remote_binaries.*` overrides when needed.
- Added a Windows-only `launchpad ssh` path which shells out to the local
  OpenSSH client, mapping resolved Launchpad SSH config onto `ssh.exe` and
  failing clearly when that client is unavailable.
- Added regression coverage for the doctor exec-environment behavior and the
  Windows SSH subprocess path.
- Updated the doctor, ssh, configuration, and troubleshooting docs to explain
  the new diagnostics behavior and the Windows interactive-shell path.
