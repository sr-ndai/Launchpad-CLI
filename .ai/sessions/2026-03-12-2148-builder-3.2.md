## Observations

- The most conservative interpretation of this task was to implement the
  status command only against `squeue` and `sacct`. Live resource utilization
  from `sstat`, remote log lookup, and cancellation remain out of scope.
- Overview mode uses `sacct` only when `--all` is requested and bounds that
  accounting query to `now-7days` to keep the default monitoring path focused
  on current and recent work.
- `--watch` intentionally rejects the root `--json` flag because a streaming
  sequence of JSON payloads is outside this prompt's required machine-readable
  scope.
