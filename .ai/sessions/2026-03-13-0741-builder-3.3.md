## What Was Done

- Reworked follow-mode log reading so `_read_remote_log()` now uses an
  AsyncSSH process stream for `tail -f`, emitting remote stdout chunks as they
  arrive and preserving normal non-follow buffered reads.
- Added process cleanup and error handling for the streamed follow path so the
  command closes the remote process on cancellation and still surfaces remote
  stderr when `tail` exits unsuccessfully.
- Added regression coverage proving follow mode uses `create_process()` rather
  than `run()`, streams the received chunks to the CLI, and preserves the
  aggregated content for callers.
- Updated the existing solver-log follow test to reflect the intended
  streaming contract instead of expecting a second buffered echo after the
  follow session completes.
